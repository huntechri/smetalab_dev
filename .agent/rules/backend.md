# Backend Engineer Rule (Next.js Server Actions / API)

## Роль и рамки
Ты — Backend-разработчик в Next.js (Route Handlers + Server Actions). Твоя зона ответственности: безопасность, RBAC, корректные API-контракты, токены/сессии, логирование, бизнес-операции.

### Обязательные принципы:
- Любой доступ к данным/действиям — только через серверную проверку прав и tenant-контекст.
- Не полагаться на UI-ограничения.
- Ошибки: структурированные, безопасные для пользователя, с детальным логированием на сервере.

## Безопасность и доступ (RBAC)
- Ревизия всех server actions и `app/api/**`: обязательные проверки прав.
- Единый паттерн: `requirePermission(permissionCode, handler)` / middleware-обёртки.
- Удаление легаси `users.isAdmin` / `users.role`; единая “точка правды”: platform role + role в команде/permissions.
- Централизация пользовательских ошибок, без утечек деталей.

## Уведомления (backend)
- `GET /api/notifications`, “mark as read”, пагинация/фильтры по необходимости.
- Запись в notifications при ключевых событиях (invite/change role/remove member и т.п.).
- (Опционально) realtime: WS/SSE/polling — серверная часть.

## Сессии и аутентификация
- **access + refresh tokens**:
  - refresh в HttpOnly cookie (ограниченный Path).
  - `/api/refresh`.
  - rotation refresh, revoke при logout.
  - корректные 401/403.

## Админка (backend)
- `/dashboard/tenants/[tenantId]`: серверные выборки (участники, приглашения, тариф/статус/метрики).
- **Имперсонация**: создание/завершение, строгая индикация, ограничения, аудит (activity logs), хранение IP инициатора.
- **Управление участниками**: смена роли, защита от снятия прав у себя, аудит `CHANGE_ROLE`.
- “Мягкое” удаление участника: `leftAt = now()` + фильтры `leftAt IS NULL`.

## Multi-tenancy (app-level)
- Гарантировать tenant-контекст во всех запросах/операциях.
- Исключить неоднозначность `tenantId IS NULL` (глобальные сущности — отдельно или явный global tenant).

## Тяжёлые операции
- Вынести embedding/долгие операции из live-потока (очередь/джобы), не блокировать транзакции.

```
┌─────────────────────────────────────────────────────────────────┐

│   Локальная БД     │                    │     Локальная БД     │

│                    ↓                    ↑                      │
│               ┌────────────┐                                   │

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Ключевые Моменты

### ✅ ПРАВИЛЬНО

```bash
# Начало дня (ВСЕГДА)
git pull
pnpm db:pull-cloud  ← Получить последние данные
pnpm dev

# Конец дня (ВСЕГДА)
pnpm db:push-cloud  ← Отправить свои изменения
git add .
git commit -m "..."
git push
```

### ❌ НЕПРАВИЛЬНО

```bash
# НЕ делай так:
pnpm dev  # ← Забыл сделать db:pull-cloud
# ... работа ...
git push  # ← Забыл сделать db:push-cloud
```

**Результат**: Другой ПК не получит твои изменения БД!

### ⚠️ КОНФЛИКТЫ

```bash
# Если забыл сделать pull:
$ pnpm db:pull-cloud
⚠️  Warning: Local changes detected. Use --force to overwrite.

# Решение 1: Сохранить локальные изменения
$ pnpm db:push-cloud  # Отправь свои изменения первым

# Решение 2: Сбросить локальные изменения
$ pnpm db:pull-cloud --force  # Перезаписать локальную БД
```

## Технические Детали

### Что происходит при push-cloud?

1. Docker контейнер создает dump локальной БД (`pg_dump`)
2. Dump отправляется в Neon (`pg_restore`)
3. Neon БД **полностью перезаписывается**

### Что происходит при pull-cloud?

1. Docker контейнер создает dump Neon БД (`pg_dump`)
2. Dump загружается в локальную БД (`pg_restore`)
3. Локальная БД **полностью перезаписывается**

### Размер данных

- **Локальная БД**: неограниченно (ограничено только диском)
- **Neon Free**: до 512MB
- **Neon Pro**: до 10GB+

### Производительность

- **Локальная БД**: ~10-50ms (localhost)
- **Neon БД**: ~100-300ms (сетевая задержка)

**Вывод**: Всегда работай с локальной БД для максимальной скорости!

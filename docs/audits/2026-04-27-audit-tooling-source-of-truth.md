# Audit tooling source of truth

This document fixes the audit/tooling drift found in the April 2026 UI audit cycle.

## Current source of truth

Button and input audits must run through the AST-based pipelines wired in `package.json`:

```bash
pnpm audit:buttons
pnpm audit:inputs
pnpm audit:ui
```

These commands delegate to:

```txt
scripts/migrate-buttons.ts --report
scripts/migrate-inputs.ts --report
```

## Generated outputs

The AST report outputs are the authoritative artifacts:

```txt
reports/button-audit.json
reports/button-canonical-spec.md
reports/input-audit.json
reports/input-canonical-spec.md
```

`docs/inputs_audit_updated.md` may still be regenerated by `scripts/migrate-inputs.ts --report` as a human-readable summary, but it is not the primary source of truth.

`docs/buttons_audit_updated.md` is a legacy generated summary and must not be treated as authoritative. Use `reports/button-audit.json` instead.

## Retired wrappers

The following legacy wrappers were removed to avoid duplicate audit entrypoints:

```txt
scripts/audit_buttons.mjs
scripts/audit_inputs.mjs
```

Reasons:

- `scripts/audit_buttons.mjs` used regex scanning and could drift from the AST pipeline.
- `scripts/audit_inputs.mjs` was only a thin wrapper around `scripts/migrate-inputs.ts --report`.
- `package.json` already exposes canonical audit commands.

## Maintenance rule

Do not add new button/input audit wrappers unless they call the canonical AST scripts and are wired through `package.json`.

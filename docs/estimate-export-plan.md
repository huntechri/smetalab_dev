# План проработки экспорта сметы в Excel и PDF

## 1) Что уже есть в текущей реализации

### UI и данные сметы
- Основная вкладка сметы рендерит `EstimateTable` и передает туда все строки сметы (`EstimateRow[]`).
- У каждой строки уже есть поля, нужные для полноценного экспорта: `kind`, `code`, `name`, `unit`, `qty`, `price`, `sum`, `expense`, а для материалов — `imageUrl`.
- В интерфейсе уже считается раздельный итог по работам и материалам.

### Экспорт сейчас
- В toolbar есть кнопка «Экспорт», но она пока не привязана к обработчику и не выполняет генерацию файла.
- Сервис `ExcelService` пока умеет только базовый `json_to_sheet`, без форматирования, изображений, формул и итоговых блоков.
- Раздел «Документы» пока заглушка; инфраструктуры под PDF-экспорт в текущем виде нет.

---

## 2) Цели по задаче

### Excel
1. Экспортировать **все данные сметы**, включая вложенные материалы под работами.
2. Оформить файл «как документ»: рамки, выравнивание, ширины колонок, шапка, группировки, freeze panes.
3. Выводить **превью материалов как изображение**, а не ссылку.
4. Добавить **раздельные итоги**: «Итого работы» и «Итого материалы».

### PDF
1. Получить печатный документ с той же структурой и итогами.
2. Поддержать изображения материалов (уменьшенные превью).
3. Сделать пригодный для отправки заказчику формат (A4, повтор заголовка таблицы на каждой странице, нумерация).

## 3) Рекомендуемая архитектура

## 3.1 Серверный сервис экспорта
Вынести логику в `lib/services/estimate-export.service.ts` (или отдельные сервисы `estimate-export-excel.service.ts` + `estimate-export-pdf.service.ts`):
- `buildEstimateExportPayload(tenantId, estimateId)` — собирает и нормализует данные для экспорта.
- `exportEstimateToExcel(payload)` — возвращает `Buffer` XLSX.
- `exportEstimateToPdf(payload)` — возвращает `Buffer` PDF.

Преимущества:
- тонкие server actions;
- единый DTO для обоих форматов;
- проще покрывать тестами.

## 3.2 DTO для экспорта
Добавить типизированный DTO (через Zod):
- meta: проект, смета, дата экспорта, валюта;
- rows: work/material в «плоском» порядке отображения;
- summary: totals `{ works, materials, grandTotal }`;
- images: карта `materialRowId -> binary/base64` (после предзагрузки).

## 3.3 Server Action + API route
- `app/actions/estimates/export.ts` (валидация входа + авторизация + вызов сервиса).
- При необходимости route для streaming download (`app/api/estimates/[id]/export/...`).

## 4) Excel: варианты реализации

## Вариант A (рекомендуется): `exceljs`
Почему:
- полноценная стилизация ячеек;
- вставка картинок в ячейки;
- merge cells, freeze panes, границы, автофильтр;
- формулы и форматы чисел.

Как реализовать:
1. Создать workbook/worksheet `Смета`.
2. Шапка документа (проект, смета, дата).
3. Таблица колонок: `Код | Наименование | Изобр. | Ед. | Кол-во | Цена | Сумма | Расход`.
4. Для каждой строки:
   - `work`: полужирный стиль + фон;
   - `material`: обычный стиль + отступ по иерархии;
   - вставка изображения в колонку `Изобр.` (если есть).
5. Итоговый блок:
   - `Итого работы`;
   - `Итого материалы`;
   - `Итого смета`.
6. Оформление:
   - border по всей таблице;
   - выравнивание по типу данных;
   - денежный формат `#,##0.00 [$₽-419]`;
   - freeze first row + autofilter.

Минусы:
- более тяжелая библиотека, чем `xlsx`.

## Вариант B: текущий `xlsx` + минимальный стиль
Можно быстро сделать плоский экспорт с итогами, но:
- стили ограничены;
- вставка изображений нестабильна/ограничена;
- итоговое качество документа будет хуже.

Вывод: для вашей цели по «аккуратному документу с изображениями» лучше переход на `exceljs`.

## 5) PDF: варианты реализации

## Вариант A (рекомендуется): HTML -> PDF (Playwright)
Пайплайн:
1. Генерируем серверный HTML шаблон сметы (таблица, стили печати, итоги).
2. Рендерим через Playwright в headless Chromium.
3. Выгружаем PDF (`page.pdf`) с `format: 'A4'`, `printBackground: true`, header/footer.

Плюсы:
- качество как у «печати страницы»;
- легко поддерживать фирменный стиль;
- удобные таблицы и переносы.

Минусы:
- runtime-зависимость от браузера.

## Вариант B: `pdfmake` / `pdf-lib`
Генерация PDF программно.

Плюсы:
- без браузера.

Минусы:
- сложнее сделать красивую верстку таблицы;
- изображение и переносы страниц потребуют больше ручной логики.

Вывод: если на проде допустим Playwright runtime, вариант A даст лучший UX и скорость разработки.

## 6) Обработка изображений материалов

1. На этапе построения payload собрать уникальные `imageUrl` материалов.
2. Серверно скачать изображения с лимитами:
   - timeout,
   - max-size,
   - только безопасные MIME.
3. При ошибке картинки — подставлять placeholder.
4. Для Excel:
   - хранить в памяти как `Buffer`;
   - вставлять в конкретную ячейку и увеличивать высоту строки.
5. Для PDF:
   - преобразовывать в data URI;
   - ограничивать размер блока (например 48x48).

Важно: не тянуть изображения на клиент ради экспорта — это медленно и нестабильно.

## 7) Предложение по UX

- В выпадающем меню «Экспорт» добавить:
  - `Экспорт в Excel (.xlsx)`
  - `Экспорт в PDF (.pdf)`
- Показать индикатор прогресса/спиннер.
- При больших сметах — фоновая задача + уведомление о готовности (этап 2).
- Ошибки по картинкам делать некритичными: файл должен скачиваться даже с частичными превью.

## 8) Тест-план

## Unit
- трансформация rows -> export DTO;
- корректность итогов по работам/материалам;
- обработка пустых/битых imageUrl;
- форматирование чисел/денег.

## Integration
- server action: авторизация + валидация;
- генерация XLSX буфера (non-empty, содержит ожидаемые колонки/листы);
- генерация PDF буфера (non-empty, валидный `%PDF`).

## E2E (опционально)
- клик по «Экспорт в Excel/PDF» -> успешная загрузка файла;
- проверка имени файла и MIME.

## 10) Конкретные рекомендации по вашему ТЗ

1. **Экспортировать все данные сметы** — использовать единый export payload из текущих `EstimateRow` + метаданные проекта/сметы.
2. **Аккуратное оформление Excel** — перейти на `exceljs`, заложить таблицу в стиле «печатной формы» с рамками и форматами.
3. **Изображения как превью** — серверно скачивать и встраивать изображения в ячейки Excel/PDF.
4. **Итоги отдельно по работам и материалам** — формировать отдельный Summary-блок внизу + при желании отдельный лист `Итоги`.
5. **PDF после Excel** — сначала стабилизировать export payload и Excel, затем переиспользовать тот же payload для PDF.

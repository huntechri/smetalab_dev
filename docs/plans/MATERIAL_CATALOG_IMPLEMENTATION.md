# План реализации: Добавление материалов из справочника через Dialog

Этот документ описывает архитектурный и технический подход к реализации справочника материалов, который будет доступен через модальное окно (Dialog) в модуле смет.

## 1. User Case (Сценарий пользователя)

**Актер:** Сметчик или РП.
**Цель:** Быстрое наполнение работы в смете необходимыми материалами из справочника.

### Основной поток:
1. **Триггер:** Пользователь нажимает на иконку настроек в строке конкретной «Работы» в таблице сметы.
2. **Действие:** Выбирает пункт меню «Добавить материал из справочника».
3. **Открытие модалки:** По центру экрана открывается `Dialog` (shadcn/ui) с заголовком «Справочник материалов». Основной контент сметы затеняется.
4. **Поиск и выбор:** Пользователь находит нужный материал через поиск или категории. Нажимает на позицию.
5. **Результат:**
   - Модалка закрывается (или остается открытой для пакетного добавления).
   - Под выбранной работой в смете появляется новая строка типа `material`.
   - Итоговые суммы сметы автоматически пересчитываются.

---

## 2. Архитектурная концепция

Реализация следует принципу разделения ответственности. Справочник материалов является независимой фичей.

### Компоненты (features/catalog):
- **`MaterialCatalogDialog.client.tsx`**: Основной контейнер, использующий `Dialog` из shadcn/ui.
- **`MaterialCatalogPicker.client.tsx`**: Внутренняя логика выбора (поиск, категории, список).
- **`MaterialCatalogList.client.tsx`**: Виртуализированный список материалов (`react-virtuoso`) для поддержки больших баз данных.

### Интеграция (features/projects/estimates):
- Состояние в `EstimateTable.client.tsx`: `{ isOpen: boolean, workId: string | null }`.
- Вызов через `columns.tsx` в выпадающем меню строки.

---

## 3. Технические детали

### API и Сервисы:
- Использование существующего или создание нового метода `EstimateRowsService.addMaterial`.
- Формирование иерархического кода: `{код_работы}.{номер_материала}` (например, `1.2`).

### UX Особенности:
- **Focus Trap:** Использование `Dialog` гарантирует фокус пользователя на выборе.
- **Responsive:** Модальное окно адаптируется под размер экрана, предоставляя больше пространства для данных материала (артикулы, цены), чем боковая панель.
- **Toast Notifications:** Подтверждение добавления без прерывания потока работы.

---

## 4. Схема взаимодействия

1. **User** -> Клик по "Добавить материал" в строке работы.
2. **EstimateTable** -> Установка `activeWorkId` и `isMaterialDialogOpen = true`.
3. **MaterialCatalogDialog** -> Монтируется, запрашивает данные материалов.
4. **User** -> Выбирает материал.
5. **Callback** -> `onSelect(material)` -> `estimatesActionRepo.addMaterial()`.
6. **UI** -> Обновление таблицы, автоматическое развертывание родительской работы.

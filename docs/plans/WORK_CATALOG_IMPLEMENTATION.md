# План реализации: Интеграция справочника работ в режим расчета (Sheet)

Этот документ описывает архитектурный и технический подход к реализации справочника работ, который будет доступен через боковую панель (Sheet) в модуле смет.

## 1. Архитектурная концепция

Реализация строится на принципе **Feature Isolation** (изоляция фич). Справочник работ выделяется в отдельный модуль `features/catalog`, который является независимым поставщиком данных для сметы.

### Основные слои:
- **`features/catalog`**: Содержит логику поиска, фильтрации и отображения справочника.
- **`features/estimates`**: Использует компонент из `catalog` и маппит (трансформирует) его данные в строки сметы.
- **`lib/domain/mappers`**: Чистые функции преобразования данных между фичами.

## 2. Структура данных и БД

### Таблица `work_items` (Справочник)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | uuid | Уникальный идентификатор |
| `section_id` | uuid | Ссылка на раздел (категорию) |
| `name` | text | Наименование работы |
| `unit` | string | Ед. измерения (м2, м.п., шт) |
| `base_price` | numeric | Базовая стоимость работы |
| `search_vector` | tsvector | Оптимизация полнотекстового поиска |

## 3. План реализации по этапам

### Этап 1: Создание фичи `features/catalog`
1. **Repository**: Реализовать `catalog.repository.ts` с методами:
   - `getSections()`: получение дерева категорий.
   - `searchWorks(query, sectionId)`: поиск по справочнику.
2. **Components**:
   - `WorkCatalogPicker.tsx`: Основной контейнер.
   - `CatalogSearch.tsx`: Поле ввода с дебаунсом.
   - `CatalogList.tsx`: Виртуализированный список работ (`react-virtuoso`).
   - `CatalogItem.tsx`: Карточка работы с кнопкой «Добавить».

### Этап 2: Интеграция в Смету (`EstimateTable.client.tsx`)
1. Добавление компонента в `SheetContent`:
   ```tsx
   <SheetContent side="right" className="w-full sm:max-w-xl p-0">
     <WorkCatalogPicker onAddWork={handleAddWorkFromCatalog} />
   </SheetContent>
   ```
2. Реализация функции-трансформера:
   - Принимает `CatalogWork`.
   - Создает `EstimateRow` с новым ID, сохраняя привязку к справочнику.

### Этап 3: UX Оптимизации
1. **Множественный выбор**: Режим «корзины», когда пользователь набирает работы и добавляет их одним нажатием.
2. **Кэширование**: Использование `TanStack Query` для того, чтобы справочник открывался мгновенно.
3. **Анимации**: Плавное «вылетание» добавленной работы из справочника в основную таблицу сметы (через `Framer Motion`).

---

## 4. Схема взаимодействия (Data Flow)

1. **User** открывает "Режим расчета".
2. **Sheet** монтирует `features/catalog/WorkCatalogPicker`.
3. **CatalogPicker** запрашивает данные из `CatalogRepository`.
4. **User** выбирает работу.
5. **CatalogPicker** вызывает `onAddWork(item)`.
6. **EstimateTable** получает `item`, вызывает `repository.addWorkToEstimate()`.
7. **Main Table** обновляется через оптимистичный UI или ревалидацию.

---

## 5. Преимущества подхода

1. **Чистота кода**: Файл `EstimateTable.client.tsx` не раздувается. Вся логика поиска и фильтрации скрыта внутри справочника.
2. **Масштабируемость**: Тот же самый `WorkCatalogPicker` можно будет использовать в мобильном приложении или других разделах системы (например, для мобильных актов).
3. **Производительность**: Использование виртуального списка позволяет работать со справочниками на 10 000+ позиций без лагов интерфейса.
